## Task 5: Lazy Load Charts Below the Fold

**Current state**: `ChartLazy.tsx` already uses `next/dynamic` with `ssr: false` to code-split chart.js. This handles the JS bundle lazy loading.

**What is NOT handled**: Viewport-based lazy rendering. If a report has 15 charts, all 15 render immediately (triggering 15 Canvas paints) even though only 2-3 are visible.

**Implementation**: Wrap chart rendering in an `IntersectionObserver`-based component.

**New file**: `frontend/src/components/LazyRender.tsx`

```tsx
// LazyRender.tsx — renders children only when scrolled into viewport
'use client';
import { useRef, useState, useEffect, type ReactNode } from 'react';

export function LazyRender({ children, placeholder, rootMargin = '200px' }: {
  children: ReactNode;
  placeholder: ReactNode;
  rootMargin?: string;
}) {
  const ref = useRef<HTMLDivElement>(null);
  const [visible, setVisible] = useState(false);

  useEffect(() => {
    const el = ref.current;
    if (!el) return;
    const observer = new IntersectionObserver(
      ([entry]) => { if (entry.isIntersecting) { setVisible(true); observer.disconnect(); } },
      { rootMargin }
    );
    observer.observe(el);
    return () => observer.disconnect();
  }, [rootMargin]);

  return <div ref={ref}>{visible ? children : placeholder}</div>;
}
```

**Integration**: In both `PlatformChartsPage.tsx` and `ClientPlatformPage.tsx`, wrap each `<Chart>` render call:

```tsx
<LazyRender placeholder={<SkeletonChart />}>
  <Chart key={chart.key} ... />
</LazyRender>
```

**Root margin `200px`**: Start loading 200px before the chart scrolls into view, so user rarely sees the skeleton.

**Trade-off analysis**:
- PRO: Significantly faster initial render for reports with 10+ charts. Canvas rendering is expensive.
- PRO: Memory savings — chart.js instances only created when needed.
- CON: Slight complexity increase, potential flash if user scrolls very fast.
- CON: PDF export must handle this — `exportPdfFromDOM` captures the DOM, so lazy-loaded charts that have not rendered will show as skeletons in PDF.

**PDF export mitigation**: Before PDF capture, force all `LazyRender` wrappers to render. Options:
1. Add a `forceVisible` prop that bypasses IntersectionObserver (triggered by PDF export flow)
2. Use a context/global flag: `window.__FORCE_RENDER_ALL = true` that LazyRender checks
3. Scroll to bottom programmatically before capture (hacky, avoid)

**Decision**: Option 1 — `forceVisible` prop. PDF export sets it via React context. Simple, explicit, reversible.
