### 5.2 — REQ-012: Verify HTML escaping in emails

**Problem**: User-supplied content (company name, platform label) is interpolated into email HTML in `monthlyReportJob.js:30-51` via template literals. If a company name contains `<script>` or HTML entities, it could break email rendering or (theoretically) execute in email clients.

**Current code** (`monthlyReportJob.js:37-38`):
```js
<h2 ...>${companyName}</h2>
<p ...>${platformLabel} havi riport — ${monthLabel}</p>
```

**Verification checklist**:
1. `companyName` — sourced from `company.name` (Prisma). Admin-set, not user-editable by clients. Low risk but should still be escaped.
2. `platformLabel` — hardcoded strings like "TikTok Organic", "Facebook Organic". Safe.
3. `monthLabel` — derived from `MONTH_NAMES[monthNum - 1]` + year. Safe.

**Action**: Add a simple `escapeHtml()` utility and wrap `companyName` in the email template. Minimal change:

```js
function escapeHtml(str) {
  return str.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;');
}
```

Apply to `companyName` in `buildEmailBody()`. The other fields are safe (hardcoded or derived from controlled data).

**Files**:
- `backend/services/monthlyReportJob.js` — `buildEmailBody()` (line ~30)

### 5.3 — REQ-013: Verify PDF DOM restore on error

**Problem**: Client-side PDF export temporarily modifies DOM element styles (width/maxWidth) for consistent A4 rendering. If `toPng()` throws, the styles must be restored.

**Current code** (`frontend/src/lib/exportPdfClient.ts:26-48`):
```ts
const originalWidth = element.style.width;
const originalMaxWidth = element.style.maxWidth;
element.style.width = '1120px';
element.style.maxWidth = '1120px';

try {
    // ... toPng() call
} finally {
    element.style.width = originalWidth;
    element.style.maxWidth = originalMaxWidth;
}
```

**Verdict**: ALREADY CORRECT. The `try/finally` block at lines 32-48 ensures DOM restoration even on error. No changes needed.

**Verification**: Manually test by:
1. Triggering PDF export on a report with many charts
2. Simulating failure (disconnect network mid-export, or add a temporary `throw` before `toPng`)
3. Confirming the page layout returns to normal after the error

**Files**:
- `frontend/src/lib/exportPdfClient.ts` — no changes, just verify
